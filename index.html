Abstraction allows us to focus on what an object does rather than how it does it. In this project, we use an abstract base class to define the contract for all interactable objects in the game world. 

// InteractableObject.cs
public abstract class InteractableObject : MonoBehaviour
{
    [SerializeField] protected GameObject interactionButton;
    protected bool playerInRange = false;

    // The abstract contract: Every interactable must implement this
    protected abstract void Interact(); 

    // Common logic shared by all interactables (UI Toggle)
    protected virtual void OnTriggerEnter2D(Collider2D collision) {
        if (collision.CompareTag("Player")) {
            playerInRange = true;
            interactionButton.SetActive(true);
        }
    }
}

Inheritance enables code reuse by allowing specialized classes to derive from a general base class. SpecificDoor and RandomDoor inherit all trigger and UI logic from InteractableObject, only implementing their unique interaction logic.

// SpecificDoor.cs
public class SpecificDoor : InteractableObject
{
    [SerializeField] private string sceneToLoad;

    // Reuses parent's trigger logic, only defines the "Interact" behavior
    protected override void Interact() {
        if (playerInRange) SceneManager.LoadScene(sceneToLoad);
    }
}

Polymorphism allows the system to treat different objects through a common interface. The UI button calls Interact() on an InteractableObject reference, and C# automatically executes the correct version based on the actual object type (e.g., a Door or a Sign).

// RandomDoor.cs - Overriding for custom behavior
protected override void Interact() {
    if (!playerInRange) return;

    // Polymorphic implementation: Randomly chooses a scene
    int choice = Random.Range(0, 2);
    string scene = (choice == 0) ? normalScene : abnormalScene;
    SceneManager.LoadScene(scene);
}

Encapsulation protects the internal state of objects. We use private and protected modifiers to hide sensitive data, and [SerializeField] to safely expose variables to the Unity Editor without breaking code integrity. 

// Protecting data via access modifiers
public class PlayerMovement : MonoBehaviour {
    [SerializeField] private float moveSpeed = 5f; // Hidden from other scripts
    private Rigidbody2D rb; // Internal state, encapsulated

    private void FixedUpdate() {
        // Only this class can modify the Rigidbody's velocity directly
        rb.linearVelocity = movement * moveSpeed;
    }
}
